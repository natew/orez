# Orez documentation

Orez is an easy, all-in-one way to run Zero. The Node runtime manages
zero-cache, its databases, replica storage, and local lifecycle from one
configuration. Orez Lite is a SQLite-native Zero server that uses the real
`@rocicorp/zero` client unchanged.

```sh
bunx orez
```

## Two promises

### Zero, made easy

The Node runtime is the core Orez experience. It packages stock Zero into one command and one configuration. Built-in admin and recovery tools cover the routine work around the stack.

### A smaller Zero server

Orez Lite implements Zero's server side through a deterministic Rust core. The
application still owns queries, mutators, authentication, and permissions.

## Recommended path

1. Run the [Node quick start](/docs/getting-started).
2. Use [PGlite/WASM](/docs/node/pglite) for the smallest local and CI setup.
3. Switch to [Embedded Postgres](/docs/node/embedded-postgres) when you need real PostgreSQL concurrency and logical replication.
4. Evaluate [Orez Lite](/docs/orez-lite) for a compact SQLite-native deployment.

## Clear boundaries

Node orchestrates `zero-cache`. Orez Lite replaces it with a deterministic
SQLite-native Zero server while the real `@rocicorp/zero` client and the same
application endpoints remain in place. Each runtime documents its server
architecture, operations, and test evidence independently.
