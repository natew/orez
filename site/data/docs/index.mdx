# Orez documentation

Orez is an easy, all-in-one way to run Zero. The Node runtime manages zero-cache, its databases, replica storage, and local lifecycle from one configuration. Orez Lite provides a SQLite-native engine for environments that need a smaller server.

```sh
bunx orez
```

## Two promises

### Zero, made easy

The Node runtime is the core Orez experience. It packages stock Zero into one command and one configuration. Built-in admin and recovery tools cover the routine work around the stack.

### Zero, anywhere

Orez Lite brings the Zero client model to SQLite through a deterministic Rust core. Native and Cloudflare hosts provide pull, push, permissions, and wake.

## Recommended path

1. Run the [Node quick start](/docs/getting-started).
2. Use [PGlite/WASM](/docs/node/pglite) for the smallest local and CI setup.
3. Switch to [Embedded Postgres](/docs/node/embedded-postgres) when you need real PostgreSQL concurrency and logical replication.
4. Evaluate [Orez Lite](/docs/orez-lite) for a compact SQLite-native deployment.

## Clear boundaries

Node orchestrates the stock Zero server. Orez Lite implements synchronization in a separate deterministic engine. Each runtime documents its architecture, operations, compatibility, and test evidence independently.
