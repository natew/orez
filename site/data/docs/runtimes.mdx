# Choose a runtime

Node is the recommended starting point. Orez Lite is a compact SQLite-native
Zero server for Cloudflare. Both use the same real `@rocicorp/zero` client,
queries, and mutators.

| Runtime                  | Status      | What runs                                               | Best for                                                        |
| ------------------------ | ----------- | ------------------------------------------------------- | --------------------------------------------------------------- |
| Node · PGlite            | Recommended | Stock zero-cache plus PGlite and WASM-compatible SQLite | Local development, CI, examples, and builds without native code |
| Node · Embedded Postgres | Recommended | Stock zero-cache plus a real embedded PostgreSQL server | Concurrency and production-like Zero testing                    |
| Orez Lite · Cloudflare   | Preview     | Rust/WASM engine inside a SQLite Durable Object         | Edge namespaces that fit the Durable Object model               |

## Node

Node runs the stock zero-cache package and coordinates the services around it. The PGlite profile makes the runner portable across local and CI environments, while Embedded Postgres provides concurrent connections and native logical replication. Orez Lite is a replacement Zero server. Every runtime uses the same real `@rocicorp/zero` client.

[Explore Node →](/docs/node)

## Orez Lite

Orez Lite replaces zero-cache with a deterministic Rust and SQLite server. The
application keeps its named ZQL queries, custom mutators, authenticated query
and mutate endpoints, and authoritative database. The current public
integration runs on Cloudflare.

[Explore Orez Lite →](/docs/orez-lite)
