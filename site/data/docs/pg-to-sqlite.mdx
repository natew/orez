# pg-to-sqlite

`pg-to-sqlite` is a standalone PostgreSQL-to-SQLite compiler. It parses input
with PostgreSQL's grammar, transforms the syntax tree with focused compatibility
passes, and emits SQLite SQL. It is useful when a system accepts PostgreSQL SQL
at its boundary but stores data in SQLite.

```sh
bun add pg-to-sqlite
```

```ts
import { compile } from 'pg-to-sqlite'

const result = compile('SELECT NOW() AS created_at')

console.log(result.sql)
console.log(result.warnings)
```

## Why compile the AST

SQL rewriting with regular expressions fails once casts, nested expressions,
quoted identifiers, arrays, or JSON operators are involved. This package uses
`pgsql-parser`, backed by the real PostgreSQL parser, and changes structured AST
nodes instead. The compiler currently has passes for:

- date, time, interval, `EXTRACT`, and `DATE_TRUNC` expressions;
- arrays, array literals, containment operators, and `unnest`;
- PostgreSQL casts and SQLite type affinity;
- JSON and JSONB operators and functions;
- table definitions, serial types, and defaults;
- inserts, conflicts, and returning clauses;
- PostgreSQL catalog and `information_schema` queries.

## API

### `compile(sql, options?)`

Returns `{sql, warnings}`. Translation is best-effort by default: unsupported or
ambiguous constructs produce warnings alongside the emitted SQL.

```ts
const { sql, warnings } = compile(source)
```

Set `strict: true` to throw `CompileError` if any pass produces a warning. The
error retains both `warnings` and the SQL available at the point of failure.

```ts
const result = compile(source, { strict: true })
```

### `compileMany(statements, options?)`

Compiles a list in order and returns one result per input statement.

```ts
const results = compileMany([
  'CREATE TABLE events (id BIGSERIAL PRIMARY KEY, payload JSONB)',
  "INSERT INTO events (payload) VALUES ('{}'::jsonb)",
])
```

## Schema-aware translation

Some rewrites are more accurate when the compiler knows column types, enum
definitions, or table columns. Supply a `SchemaInfo` implementation through
`options.schema`:

```ts
const result = compile(source, {
  schema: {
    getColumnType(schema, table, column) {
      return column === 'payload' ? 'jsonb' : undefined
    },
    getEnum() {
      return undefined
    },
    isEnumValue() {
      return false
    },
    getTableColumns() {
      return undefined
    },
  },
})
```

Without schema metadata the passes degrade to syntax-based inference rather
than inventing types.

## Guarantees and limits

This is a compatibility compiler, not a PostgreSQL emulator. PostgreSQL and
SQLite differ in types, concurrency, constraints, catalogs, extensions, and
function semantics; some statements cannot be made equivalent by rewriting.
Warnings are part of the public result for that reason. Use strict mode at trust
boundaries and run the emitted SQL against the SQLite build you deploy.

The test suite includes fixed translation fixtures and differential cases
against an independent PostgreSQL-on-SQLite implementation. That provides a
stronger signal than snapshot text alone, but it does not make unsupported
PostgreSQL extensions portable.
