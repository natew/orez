# Limitations

Orez Lite is a production-oriented preview with a defined synchronization surface and namespace-oriented scaling model.

## Protocol scope

The supported surface covers pull, push, mutation IDs, snapshots, incremental diffs, query-aware membership, row visibility, retention, and wake. Applications should qualify every client feature they plan to ship against the pinned Orez Lite and Zero versions.

## Host capabilities

The native host currently supports local mutators and per-namespace SQLite files. The Cloudflare host also supports delegated pushes, upstream feed ingest, platform alarms, durable writer fencing, and additional operator controls.

## Single-writer namespaces

One namespace has one ordered SQLite writer. This produces simple transaction ordering and a clear consistency boundary. It also sets the write-throughput ceiling for a single namespace. Very hot shared datasets and bulk ingest need careful partitioning and measurement.

## Retention and snapshots

Incremental history is bounded. Clients below the retained floor recover with a full snapshot. Large snapshots consume storage reads, network bytes, client apply time, and memory, so retention should reflect real offline periods and dataset sizes.

## Permission cost

Baseline row visibility uses snapshots because a rule change can revoke untouched rows. Query-aware membership offers incremental result tracking with additional stored query state, transform versioning, and server authorization logic.

## Cloudflare constraints

The Cloudflare host operates within Durable Object memory, CPU, SQLite, WebSocket, storage, and billing limits. Indexes, foreign keys, cascades, and sync metadata can amplify physical rows written. Horizontal scale comes from partitioning data into namespaces.

## Operational maturity

Packaging, native/delegated capability alignment, upgrade policy, and public compatibility guarantees are still settling. Pin versions, keep recoverable snapshots, qualify the exact host and schema, monitor namespace size and write rate, and rehearse writer fencing before launch.
