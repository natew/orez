# Orez Lite

<Status tone="preview">Preview</Status>

Orez Lite is a server for Zero. It speaks the Zero protocol to the real
`@rocicorp/zero` client and replaces Zero's server-side sync machinery.
Orez provides the small `orez/client` transport plugin needed to connect the
stock client. There is no client fork, query model, or mutator model to adopt.

Your schema, named queries, custom mutators, optimistic updates, authentication,
and application query and mutate endpoints remain ordinary Zero. Install and
use the client exactly as the [Zero documentation](https://zero.rocicorp.dev/docs)
describes, install the [`orez/client` transport](/docs/client), then point it at
the Orez Lite server instead of `zero-cache`.

<VerifiedBuildCard />

## What Orez Lite replaces

Orez Lite replaces `zero-cache` and the server infrastructure behind it with a
Rust sync engine backed by SQLite. The current public host runs as a Cloudflare
Worker and SQLite Durable Object.

```text
real @rocicorp/zero client
  │ Zero protocol
  ▼
Orez Lite server
  ├── application query and mutate endpoints
  └── authoritative database change feed
```

The server owns cookies, mutation acknowledgements, replication cursors, query
membership, row patches, retention, and wake notifications. Your application
continues to own its data model, authentication, permissions, business logic,
and authoritative database.

## What changes in an application

Only the server deployment and address change. Client components continue to
use the same Zero queries, mutators, hooks, and local cache. Orez-specific code
belongs at the server hosting boundary.

For client APIs and application endpoint implementation, use Zero's own docs:

- [Install Zero](https://zero.rocicorp.dev/docs/install)
- [Queries](https://zero.rocicorp.dev/docs/queries)
- [Mutators](https://zero.rocicorp.dev/docs/mutators)
- [Authentication](https://zero.rocicorp.dev/docs/auth)
