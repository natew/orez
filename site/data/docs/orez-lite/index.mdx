# Orez Lite

<Status tone="preview">Preview</Status>

Orez Lite is a SQLite-native synchronization engine for Zero applications. It
keeps the application-facing Zero model intact: named ZQL queries, custom
mutators, optimistic local state, authenticated query and mutate endpoints, and
one authoritative application database.

Orez Lite replaces the sync engine behind those endpoints. Application code
continues to use the stock `@rocicorp/zero` client or `on-zero` helpers.

<VerifiedBuildCard />

## What stays the same

- Define and call named queries with Zero or `on-zero`.
- Define custom mutators once and run them optimistically on the client and
  authoritatively on the server.
- Authenticate the application query and mutate endpoints.
- Express read access in authenticated ZQL.
- Enforce write access in server-side mutators.
- Keep the application database as the write authority.

Zero’s documentation remains the source of truth for these APIs:

- [Install and set up Zero](https://zero.rocicorp.dev/docs/install)
- [Queries](https://zero.rocicorp.dev/docs/queries)
- [Mutators](https://zero.rocicorp.dev/docs/mutators)
- [Authentication and permission patterns](https://zero.rocicorp.dev/docs/auth)
- [ZQL](https://zero.rocicorp.dev/docs/zql)

## What Orez Lite changes

Orez Lite replaces `zero-cache`, PostgreSQL logical replication, and the CVR
implementation with a Rust sync engine backed by SQLite. The current public host
runs as a Cloudflare Worker and SQLite Durable Object.

```text
Zero / on-zero application
  │ named queries · custom mutators · auth
  ▼
application query + mutate endpoints
  │ authoritative database commits
  ▼
Orez Lite replication + query tracking
  │ pull · patches · wake
  ▼
stock Zero client cache
```

The engine owns sync bookkeeping: cookies, mutation acknowledgements,
replication cursors, query membership, row patches, retention, and wake
notifications. It does not define application permissions or business logic.
