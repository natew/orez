# What stays Zero

Orez applications keep Zero’s client and application APIs. The runtime choice
changes the server machinery behind them.

## Application compatibility

| Surface                        | Node runtime                     | Orez Lite                             |
| ------------------------------ | -------------------------------- | ------------------------------------- |
| `@rocicorp/zero` client        | Stock                            | Stock, using `on-zero` HTTP transport |
| Named queries and ZQL          | Stock                            | Unchanged                             |
| Custom mutators                | Stock                            | Unchanged                             |
| Query endpoint                 | Application-owned                | Same application endpoint             |
| Mutate endpoint                | Application-owned                | Same application endpoint             |
| Authentication and permissions | Application queries and mutators | Unchanged                             |
| Sync engine                    | Stock `zero-cache`               | Orez Lite Rust/SQLite engine          |
| Change source                  | PostgreSQL logical replication   | Application database change feed      |

Orez Lite does not add an application permission language or a second mutator
registry. Read access remains authenticated ZQL from the query endpoint. Write
access remains server-side mutator logic from the mutate endpoint.

## Client transport

The current Orez Lite host exposes pull, push, and wake over HTTP rather than
the stock `zero-cache` WebSocket. `on-zero` adapts that transport to the stock
Zero client, so components continue using the same queries, mutations, and
reactive local cache.

## Server implementation

The Node runtime runs stock `zero-cache`. Orez Lite implements the sync side
with its own cookies, mutation acknowledgements, query membership, retention,
and row patches. Those internals are qualified against real Zero clients and
observable application behavior.

<Callout title="Versioning boundary">
  Pin and qualify the exact Orez, `on-zero`, and `@rocicorp/zero` versions used by an
  application. Orez’s transport follows Zero’s wire protocol, whose internal details can
  change between Zero releases.
</Callout>

For application APIs, use Zero’s official [Queries](https://zero.rocicorp.dev/docs/queries),
[Mutators](https://zero.rocicorp.dev/docs/mutators),
[Authentication](https://zero.rocicorp.dev/docs/auth), and
[ZQL](https://zero.rocicorp.dev/docs/zql) documentation.
