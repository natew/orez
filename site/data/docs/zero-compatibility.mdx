# The real Zero client, unchanged

Orez does not provide a client-side reimplementation of Zero. Applications use
the real `@rocicorp/zero` package, its local cache, ZQL, named queries, custom
mutators, framework integrations, and optimistic updates.

The difference between the Orez runtimes is entirely on the server:

| Surface                                | Node runtime                     | Orez Lite                                      |
| -------------------------------------- | -------------------------------- | ---------------------------------------------- |
| Browser package                        | `@rocicorp/zero`                 | `@rocicorp/zero`                               |
| Client API and local cache             | Zero                             | Zero                                           |
| Schema, named queries, and mutators    | Zero                             | Zero                                           |
| Application query and mutate endpoints | Zero                             | Zero                                           |
| Sync protocol                          | Zero protocol                    | Zero protocol                                  |
| Server implementation                  | `zero-cache` coordinated by Orez | Orez Lite SQLite-native server                 |
| Authoritative data                     | Postgres or PGlite               | Application database replicated into Orez Lite |

## What an application changes

For Node, point the client at the `zero-cache` address printed by Orez. For
Orez Lite, point that same client at the Orez Lite server. There is no second
client dependency and no Orez-specific query or mutation API.

Zero's documentation is the source of truth for application development:

- [Install Zero](https://zero.rocicorp.dev/docs/install)
- [Queries](https://zero.rocicorp.dev/docs/queries)
- [Mutators](https://zero.rocicorp.dev/docs/mutators)
- [Authentication](https://zero.rocicorp.dev/docs/auth)

Orez documents the server concerns that differ: deployment, SQLite storage,
replication, retention, recovery, operational controls, and the exact Zero
versions qualified by its conformance suites.
